# Frontend Design Migration Guide

## 1. 文档定位

本文件用于样式治理的**执行手册**：阶段目标、批次顺序、验收口径、风险与回滚。

## 2. 基线输入

- 审计文档：`frontend/docs/ui-style-audit-2026-04-11.md`
- 长期规范：`frontend/DESIGN-FOUNDATIONS.md`

## 3. 主干口径

- 发布判定以主干运行页面为准
- `backup` / `copy` / 实验文件不计入阶段完成标准

## 4. 分阶段执行

### 阶段 0：冻结增量

- 建立巡检与门禁：
  - `npm run style:report`（告警）
  - `npm run style:gate`（阻断）
- 规则覆盖：硬编码颜色、非标准字号、Modal `bodyStyle`
- 验收：新增违规为 0

### 阶段 1：重点页面治理

优先顺序：

1. `MainLayout`
2. `FileList` / `DocumentUpload`
3. `ProjectDatasetView`
4. `PatientDetail`

统一目标：

- 颜色收敛到 token/CSS 语义变量
- 字号收敛到 12/14/16/20/24
- Modal 收敛到 4 档 + `styles.body`

### 阶段 2：自动化固化

- 规则默认阻断
- 固化通用封装（如 Modal confirm）
- 收敛多来源样式常量到 SSOT

## 5. 一期重点页面最小批次（FileList / DocumentUpload）

### Batch FL-1

- 范围：筛选区、状态标签、基础文本层级
- 验证：筛选/分页/检索正常，状态语义无错色

### Batch FL-2

- 范围：表格文本样式、详情弹层入口样式
- 验证：表格交互正常，弹层滚动与显示正常

### Batch DU-1

- 范围：统计卡与上传区壳层
- 验证：上传入口、拖拽、批量选择不受影响

### Batch DU-2

- 范围：设置/预览弹窗、解析列表样式
- 验证：轮询刷新与删除/解析/审核流程可用

### Batch FLDU-3

- 范围：收口与例外清理
- 验证：新增改动区域违规为 0

## 6. 验收标准

### 定量

- P0 页面硬编码颜色下降 80%+
- 非标准字号下降 90% 左右
- Modal 宽度由 12 档收敛到 4 档
- `FileList/DocumentUpload` 新增改动区域违规为 0

### 定性

- 替换可追溯（每批提供 `old -> token` 映射清单）
- 关键页面视觉一致，无语义色漂移

## 7. 回滚策略

- 每批次独立提交，按批次回滚
- 回滚触发：
  - 功能回归
  - 视觉错配影响语义理解
  - 规则误报导致主线阻断
